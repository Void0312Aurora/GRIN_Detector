## 2. 实验目的

本实验的目的如下：

1. 完成 qDIC 原型系统的基本搭建、对光和初始成像。
2. 验证 qDIC 原型系统是否具备稳定、可重复的成像能力。
3. 使用 qDIC 采集微透镜在 `clean` 与 `defect` 状态下的真实观测数据。
4. 验证同一颗微透镜在重复采集条件下，qDIC 结果是否具有足够稳定性。
5. 验证 `clean -> defect` 的差异是否能够在 qDIC 图像中被稳定观察和局部化。
6. 为当前项目的真实数据验证、QC / gating 分析以及后续实验设计提供基础数据。

## 3. 实验原理

### 3.1 为什么“系统搭建”也是实验的一部分

如果实验室没有现成 qDIC，那么“把样品放上去采图”并不是实验的起点。  
真正的实验起点应该是：

- 光学路径是否搭起来了
- 偏振 / 剪切 / 成像链是否工作
- 系统是否能稳定输出可解释的 qDIC 图像

因此，本报告不应只记录样品采集，还应记录：

- qDIC 系统结构
- 关键部件选型
- 光路装调过程
- 初始验收结果

### 3.2 qDIC 原理

qDIC（Quantitative Differential Interference Contrast）利用相位梯度或光程差导致的强度变化形成对比，可用于观察微透镜表面局部结构变化、边缘扰动和缺陷引起的微小差异。

在本实验中，qDIC 是唯一保留的观测模态，承担以下角色：

- 提供真实观测输入。
- 用于分析同一颗微透镜在 `clean / defect` 状态下的图像差异。
- 用于评估重复采集稳定性、图像质量与异常样本。

### 3.3 qDIC 原型系统的最小组成

如果是自建或半自建 qDIC，最少需要明确以下组成模块：

- 光源模块
- 偏振模块
- 照明 / 聚光模块
- 剪切或 DIC 对比形成模块
- 物镜与成像模块
- 分析器 / 检偏模块
- 管镜与相机模块
- 样品台与定位模块
- 采集软件与数据保存模块

这里不要求这一步就把系统做到“论文级定量精度”，但至少要先做到：

1. 能稳定成像
2. 图像可重复
3. 同一视场的变化具有可解释性

### 3.4 与当前项目主线的关系

当前 `mini_grin_rebuild` 的主线依赖“平滑标准面 + 稀疏小缺陷”的结构假设。  
在 qDIC-only 条件下，本实验主要能够回答以下问题：

1. 同一颗 clean 微透镜的重复采集是否稳定。
2. 同一颗微透镜从 `clean` 到 `defect` 的变化是否可在 qDIC 中稳定显现。
3. 当前 QC / gating 思路是否能区分正常采集与异常采集。

### 3.5 当前版本的边界

由于本实验暂不包含 WLI，因此当前结论有明确边界：

- 不能直接获得三维形貌参考。
- 不能直接验证缺陷是否满足“稀疏小残差”的几何定义。
- 不能直接构造严格意义上的弱 GT。

因此，本报告的重点应放在：

- qDIC 重复性
- clean / defect 差分可见性
- 实验稳定性
- 失败模式与后续改进建议

## 5. 实验步骤

### 5.1 qDIC 方案确认与部件准备

1. 明确当前 qDIC 方案是商用整机、附件改装，还是自建原型。
2. 列出最小可用部件清单，确认哪些已经到位，哪些仍缺失。
3. 确认光路结构、机械安装方式和相机接口。
4. 确认本次实验的目标是“先跑通成像”还是“进入微透镜采集”。
5. 记录系统版本、光路示意和当天环境条件。

### 5.2 光学与机械组装

1. 安装光源、偏振元件、成像镜组、相机和样品台。
2. 固定光学元件位置，减少机械晃动与角度漂移。
3. 安装并检查剪切 / DIC 模块或相应差分相衬元件。
4. 建立从光源到相机的基本通光路径。
5. 记录各元件型号、安装顺序和相对位置。

### 5.3 初始对光与成像跑通

1. 在无样品或简单样品条件下检查是否有稳定图像输出。
2. 调整焦距、照明强度、曝光和相机增益，使图像进入可用范围。
3. 检查视场均匀性、是否存在明显遮挡、条纹、失焦或饱和。
4. 获取暗场、空场或平场图像，作为后续校正基础。
5. 若此阶段无法获得稳定图像，则不要继续样品采集。

### 5.4 qDIC 工作点调整与最小验收

1. 使用简单结构样品或规则样品测试 qDIC 对比是否正常。
2. 调整偏振、分析器、剪切相关设置，使图像对局部结构变化有稳定响应。
3. 对同一视场重复采集多次，检查重复性。
4. 记录最小验收结果：
   - 是否稳定出图
   - 是否重复性可接受
   - 是否能看到预期结构对比
   - 是否能完成数据保存
5. 只有通过最小验收后，才进入微透镜样品实验。

### 5.5 clean 状态采集

1. 在低倍视场下确认目标微透镜位置。
2. 切换到目标采集视场，保证视场中主要只有一颗微透镜，且 aperture 尽量完整。
3. 对同一颗 clean 微透镜做重复采集。
4. 记录每次采集的 repeat 编号、原始文件路径、stage 信息和备注。

### 5.6 defect 状态采集

1. 对目标样品引入可控缺陷，或选择已有缺陷样品。
2. 尽量回到与 clean 状态一致的视场与参数。
3. 对 defect 状态进行重复采集。
4. 记录缺陷类型、位置、是否可逆以及采集备注。

### 5.7 实验后检查

1. 检查原始文件是否完整且可读。
2. 检查 `clean / defect` 标记是否清晰。
3. 检查同一颗微透镜的重复采集是否出现明显漂移。
4. 检查是否存在过曝、饱和、遮挡、边界截断或失焦。
5. 检查本次系统状态是否相比验收阶段发生明显漂移。

## 7. 数据处理

### 7.1 系统验收数据整理

1. 整理暗场、空场、平场以及对光 / 验收样品图像。
2. 记录每轮调整对应的参数设置和图像结果。
3. 标记“可用工作点”和“失败工作点”，避免后续重复试错。

### 7.2 原始数据整理

1. 导入原始 qDIC 数据。
2. 按 `sample_id / clean / defect / repeat` 重命名或归档。
3. 核对元数据、采集参数与原始图像是否一致。

### 7.3 图像预处理

1. 进行必要的平场、暗场和坏点处理。
2. 对图像强度做统一归一化。
3. 裁剪单颗微透镜区域，去除无关背景。
4. 标记有效 aperture 区域。

### 7.4 重复性分析

1. 对同一颗 clean 微透镜的 repeats 做配准。
2. 计算 clean-clean 之间的差异图。
3. 统计重复采集之间的均值差、方差或其他稳定性指标。

### 7.5 clean / defect 差异分析

1. 对同一颗微透镜的 clean 与 defect 图像做配准。
2. 计算 `defect - clean` 差分图。
3. 观察差异是否局部集中、是否可重复出现。
4. 判断差异是否可能由缺陷引起，或更可能由漂移、照明变化或失焦引起。

### 7.6 QC / 异常筛查

建议至少检查：

- 是否过曝或饱和
- 是否失焦
- 是否发生大尺度平移或旋转漂移
- 是否存在 aperture 截断
- 是否有明显照明不一致

## 9. 误差分析

本实验可能的误差来源包括：

1. **系统装调误差**：光学元件安装不正或位置不稳会直接破坏对比质量。
2. **偏振 / 剪切工作点不稳定**：会导致图像对比不可重复。
3. **光路漂移**：重复采集间的系统漂移会引入伪差分。
4. **焦平面变化**：轻微失焦可能被误认为缺陷变化。
5. **照明不稳定**：强度变化会影响差分结果的可解释性。
6. **样品位置偏差**：视场未能严格回到同一位置时，会放大边缘差异。
7. **边界截断**：aperture 不完整时，边缘伪影会污染分析。
8. **缺乏形貌参考**：由于当前不包含 WLI，无法直接判断 qDIC 差异是否对应真实几何缺陷。

## 10. 问题讨论

### 10.1 当前 qDIC-only 方案能回答什么

当前方案更适合回答：

- qDIC 原型系统是否已经具备最小可用性
- 重复采集是否稳定
- clean / defect 差分是否可见
- QC 是否能筛出异常采集

### 10.2 当前 qDIC-only 方案不能直接回答什么

当前方案暂时不能直接回答：

- 缺陷是否满足“稀疏小残差”几何假设
- 差异是否与真实高度残差严格对应
- 当前主线中的物理假设在真实几何上是否完全成立
